How the repository is organized

Source, Adapters, Hooks

The repository is public source, not a runtime cache. Package source stays in packages/, generated Codex output stays under .agents/plugins/generated/, and shared tooling stays in bin/ and hooks/.

packages/<plugin>/

Canonical source for plugin metadata, skills, hooks, CLIs, README files, and changelogs.

.agents/plugins/generated/

Generated Codex adapter packages produced from the source package layout.

bin/plugin-adapters

Build, check, and diff generated runtime forms before shipping marketplace changes.

bin/plugin-versions

Checks or writes package versions using the repository's version convention.

hooks/

Repository-level git hooks that keep versions, adapters, and commit metadata disciplined.

docs/

GitHub Pages source, migration documentation, and public site assets.

Use it

Installation

Add the marketplace once, then install the packages that match the agent runtime and workflow you use.

Claude Code

claude plugins marketplace add epologee/laicluse-agent-fieldkit
claude plugins install git-discipline@laicluse-agent-fieldkit
claude plugins install rover@laicluse-agent-fieldkit gurus@laicluse-agent-fieldkit

Codex

codex plugin marketplace add epologee/laicluse-agent-fieldkit
codex plugin add git-discipline@laicluse-agent-fieldkit
codex plugin add rover@laicluse-agent-fieldkit
codex plugin add gurus@laicluse-agent-fieldkit

For local development, use claude plugins marketplace add ./ or codex plugin marketplace add ./ from the repository root.
